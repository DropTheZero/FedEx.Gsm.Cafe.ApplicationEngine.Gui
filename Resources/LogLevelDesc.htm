Level Usage
Critical (1) For information that must be logged. This level will always be logged.
Error (2) For software errors: edit or unexpected. Most likely will accompany displaying an error to the user. The displayed error can refer the user to the log for more information.
Warning (3) For informational purposes. Non-error conditions but something that might affect something or would be helpful for debugging.
Info (4) Informational only. Non-error and non-warning conditions.
Verbose (5) Informational only. One more level of non-error and non-warning informational logging.
Debug (6) Set for full logging. All levels will be logged. Do not leave anything at this level in production unless you are trying to debug a problem.
